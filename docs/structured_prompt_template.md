# 结构化 Prompt 模板

本文档定义结构化任务源文件的固定格式。结构化源文件用于记录任务变量、精确参数和验收要点；流水线当前仍读取展开后的 `prompt.txt`。

## 使用原则

- `可选值` 描述该维度允许的实验变量空间。
- `当前选择` 表示本次具体任务采用的选项。
- `精确参数` 必须足以无歧义地生成完整 `prompt.txt`。
- 每个结构化文件只描述一个具体任务实例。
- `Prompt展开.章节` 必须覆盖现有 prompt contract 要求的章节。

## 与 prompt.txt 的映射

| 结构化字段 | 展开到 prompt.txt |
|---|---|
| `游戏` | `任务身份` |
| `运行契约` | `输出契约` |
| `世界规则` 和 `情境维度.网格大小` | `坐标与时间` |
| `视觉.颜色` | `颜色表` |
| `实体` 和 `情境维度` | `布局与实体` |
| `控制` | `输入` |
| `更新顺序` | `更新顺序` |
| `核心规则` | `独特规则` |
| `计分` 和 `终止` | `计分与终止` |
| `视觉.绘制顺序`、`视觉.HUD`、`视觉.终局显示` | `绘制顺序与 HUD` |
| `禁止项` | `禁止项` |
| `验收检查` | `提交前检查` |

## 固定模板

```yaml
规范版本: "1.0"

游戏:
  标识: <游戏ID>
  基础类型: <基础游戏类型>
  任务名称: <独立任务名称>
  难度: <简单|中等|困难>
  任务描述: <一句话描述核心玩法>
  去模板化要求: <禁止使用经典默认规则替代的说明>

情境维度:
  <维度名称>:
    可选值: [<选项一>, <选项二>, <选项三>]
    当前选择: <选项一>
    精确参数:
      <参数名称>: <精确值>

运行契约:
  编程语言: Python 3
  游戏框架: pygame
  单文件: true
  允许外部资源: false
  窗口尺寸: [800, 600]
  窗口标题: <任务名称>
  帧率: 60
  随机种子: 42
  重开按键: R
  退出按键: ESC

世界规则:
  坐标原点: 左上角
  游戏区域: [<x>, <y>, <宽度>, <高度>]
  碰撞方式: <网格重合|pygame矩形|圆心距离>
  时间单位: 毫秒
  状态更新间隔: <毫秒数>
  坐标换算: <网格坐标到像素坐标的公式>

实体:
  玩家:
    形状: <形状>
    尺寸: <尺寸>
    初始位置: <坐标>
    初始状态:
      <状态名称>: <初始值>

  游戏对象:
    - 标识: <对象ID>
      类型: <收集物|敌人|障碍|目标>
      尺寸: <尺寸>
      初始状态: {}
      生成规则: <精确规则>
      移动规则: <精确规则>
      消失规则: <精确规则>

控制:
  - 输入: <按键或鼠标操作>
    动作: <动作名称>
    状态影响: <具体改变的状态>
    限制:
      - <限制条件>

核心规则:
  - 编号: REQ-01
    名称: <规则名称>
    触发条件: <精确触发条件>
    前置条件:
      - <条件>
    状态变化:
      - <变量及变化方式>
    视觉反馈:
      - <可见反馈>
    分数影响: <数值或无>
    终止影响: <胜利|失败|无>

计分:
  初始分数: 0
  计分事件:
    - 事件: <事件名称>
      分数变化: <整数>

终止:
  胜利条件:
    - <条件>
  失败条件:
    - <条件>
  终局冻结游戏: true
  重开恢复全部状态: true

视觉:
  颜色:
    背景: [0, 0, 0]
    文字: [255, 255, 255]
    玩家: [<R>, <G>, <B>]
    目标: [<R>, <G>, <B>]
    危险对象: [<R>, <G>, <B>]

  绘制顺序:
    - 背景
    - 游戏区域
    - 静态对象
    - 动态对象
    - 玩家
    - HUD
    - 终局覆盖层

  HUD:
    字号: 28
    项目:
      - 文本: "<文本模板>"
        位置: [<x>, <y>]

  终局显示:
    失败文本: Game Over
    胜利文本: You Win
    重开文本: Press R to Restart
    位置: [<x>, <y>]

更新顺序:
  - 处理事件
  - 应用输入
  - 更新时间与冷却
  - 更新玩家
  - 更新其他实体
  - 处理碰撞与交互
  - 更新分数和进度
  - 检查胜负
  - 绘制画面
  - 刷新窗口
  - 限制帧率

禁止项:
  - 外部资源
  - 未声明的随机行为
  - 额外游戏模式
  - <游戏特定禁止行为>

验收检查:
  可执行性:
    - pygame已初始化
    - 成功创建窗口
    - 存在事件循环
    - 使用时钟限制帧率
    - 支持正常退出

  功能性:
    - 输入真实改变状态
    - 游戏状态持续演化
    - 核心规则形成闭环
    - 计分和目标具有可见反馈
    - 胜利或失败状态可以到达

  体验:
    - 重要对象视觉可区分
    - HUD信息完整
    - 终局反馈清晰
    - 终局后状态冻结
    - 不存在无限忙循环

Prompt展开:
  章节:
    - 任务身份
    - 输出契约
    - 坐标与时间
    - 颜色表
    - 布局与实体
    - 输入
    - 更新顺序
    - 独特规则
    - 计分与终止
    - 绘制顺序与 HUD
    - 禁止项
    - 提交前检查
```
