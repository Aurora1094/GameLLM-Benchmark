你是一位资深 Python 游戏开发者，精通 pygame。
你的任务是生成一个**单文件、可直接运行**的 Pong 游戏。

# 运行要求
你的代码必须能作为独立 Python 脚本直接运行：
- 代码必须是完整 Python 源码，语法正确，解释器可以直接解析并执行。
- 只使用标准库和 pygame，必须正确导入 pygame 并初始化游戏所需依赖。
- 创建一个 800x600 像素的 pygame 图形窗口，并在运行过程中持续渲染游戏画面。
- 建立完整的事件循环，能够持续接收玩家输入和窗口事件。
- 程序启动后应在短时间内稳定进入游戏流程，不因初始化、资源或主循环错误崩溃，也不应提前退出。
- 主循环必须可控，使用固定帧率或等效节奏控制，避免失控忙等待，并能响应窗口关闭请求后干净退出。

# 游戏规格
经典乒乓对战：两块竖直球拍分居屏幕左右，一颗球在中间往返。
玩家控制左侧球拍，目标是让球越过对手一侧得分，先达到目标分数者获胜。

运行参数：窗口尺寸 800x600，主角颜色 #FFFFFF，单局时长 120 秒。

你必须实现以下全部功能，每一条都要在游戏中真实生效：
1. 玩家用上下方向键控制己方球拍上下移动，球拍不超出屏幕边界
2. 球碰到上下墙壁和球拍时正确反弹，方向与速度变化合理
3. 球越过一侧边界时对方得分，并在画面上实时显示双方比分
4. 对手球拍能自动追踪球的位置进行防守，构成可玩的对抗
5. 任一方达到目标分数时本局结束，并显示胜负结果

# 代码与体验
在满足上述功能的前提下，请按以下顺序组织代码质量：
- 控制复杂度，避免过长函数、过深嵌套和难以维护的单一大循环。
- 减少重复代码，把反复出现的逻辑抽成函数或类，保持规则实现一致。
- 常量集中管理，将窗口、颜色、速度、尺寸、坐标、间距、字体大小、时间、分数阈值等参数定义为有语义的常量，避免同类参数散落在规则、更新和绘制逻辑中。
- 命名规范可读，变量、函数、类和参数名称应体现其游戏语义。
- 模块划分清晰，把初始化、输入处理、状态更新、碰撞/规则判断、绘制和重置等职责分开。
- 注释适度，优先解释关键规则、状态转换和不易直接读懂的逻辑。
同时为关键的游戏状态（如得分、时间、生命、胜负或游戏结束）提供玩家可见的反馈。
