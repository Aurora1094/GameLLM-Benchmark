请使用 Python 3 和 pygame 实现一个可直接运行的 Dodge Blocks Easy 游戏，并严格遵守以下固定规范，以保证不同模型输出的稳定性和可比性。

通用约束：
1. 只输出完整 Python 代码，不要输出解释、Markdown、代码块标记。
2. 代码必须单文件可运行，不能依赖任何本地图片、音频、字体或其他外部资源。
3. 使用 `pygame`，窗口大小固定为 `800x600`，帧率固定为 `60 FPS`。
4. 所有可调参数都写成顶部常量，随机行为使用 `random.seed(42)`。
5. 必须支持关闭窗口退出、`R` 重新开始、`ESC` 退出。
6. 必须显示存活时间或分数，以及游戏结束提示。

玩法与布局固定要求：
1. 玩家角色是底部可左右移动的方块，大小固定为 `60x60` 像素，初始位于屏幕底部中央，距离底边约 `40` 像素。
2. 玩家仅能左右移动，控制方式固定为 `左/右方向键` 或 `A/D`，移动速度固定为每帧 `7` 像素。
3. 障碍物为从屏幕顶部落下的方块，大小固定为 `40x40` 像素。
4. 每个障碍物的下落速度在 `4` 到 `8` 像素/帧之间，但必须由固定随机种子生成。
5. 每 `40` 帧生成一个新障碍物，横向生成位置随机，但必须完整出现在屏幕内。
6. 玩家与任意障碍物发生矩形碰撞时立即游戏结束。
7. 分数规则固定为按存活时间累计，建议每存活 `1` 秒加 `1` 分，并实时显示。
8. 障碍物落出屏幕后应被移除，避免内存和性能问题。
9. 游戏结束后显示 `Game Over`、最终分数和 `Press R to Restart`。

视觉要求：
1. 背景、玩家、障碍物、HUD 颜色要明显区分。
2. 不要加入射击、护盾、道具、多生命或关卡系统。
